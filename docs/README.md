## OpenAPI의 역할
#### 증권사가 외부 프로그램에서 자기 시스템과 연결할 수 있도록 제공하는 인터페이스
증권사 HTS/MTS 대신 **시세 조회, 계좌 조회, 주문 요청**

**C:\OpenAPI\KOAStudioSA.exe**에 개발을 위한 함수 목록 존재  

OpenAPI는 (함수나 이벤트 모두) 스레드를 지원하지 않으므로  OpenAPI함수나 이벤트를 스레드에서 호출하는것은 안정성을 보장 ㅌ
## 주요 용어
### 로그인 / 인증

증권사 API를 사용하려면 먼저 로그인하고 권한을 확인

### API Key / App Key / Secret Key

일부 증권사 REST API에서 사용하는 인증 정보

### Access Token

로그인 후 일정 시간 동안 API를 호출할 수 있게 해주는 임시 인증값

### 세션(Session)

로그인 이후 유지되는 연결 상태

### 요청(Request)

내 프로그램이 증권사 서버에 기능을 요청

### 응답(Response)

요청에 대한 서버의 결과

### 실시간 데이터 / 실시간 시세

서버가 가격 변화를 계속 밀어주는 방식

### 조회 API

정보를 읽어오는 기능

### 주문 API

실제로 매수/매도 요청을 보내는 기능
